Here’s a minimal example of how you can include screenshots in your README.md. Put your image files in a folder (e.g. screenshots/) next to your README, then reference them like this:

markdown
# My Project

Brief project description…

## Screenshots

### Home Page
![Home page screenshot](./screenshots/home-page.png)

### Settings Panel
![Settings panel screenshot](./screenshots/settings-panel.png)

### Mobile View (smaller width)
<img src="./screenshots/mobile-view.png" alt="Mobile view screenshot" width="300"/>

Notes:

  1. Folder structure

    arduino
    your-repo/
├── README.md
└── screenshots/
    ├── home-page.png
    ├── settings-panel.png
    └── mobile-view.png

  2. Markdown syntax

    • ![Alt text](path/to/image.png) is the most common way.

    • If you need to tweak size, you can drop into HTML:

      html
      <img src="./screenshots/mobile-view.png" alt="Mobile view" width="300"/>

  3. Relative paths

    • Using ./screenshots/... makes sure GitHub (or other renderers) picks up the images correctly.

    • If you host images elsewhere, you can also use full URLs:

      markdown
      ![Hosted image](https://example.com/path/to/screenshot.png)

  4. Alt text

    • Always include a descriptive alt text for accessibility.

This should get your screenshots showing up nicely in your project’s README.
